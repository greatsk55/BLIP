# BLIP

**話して、消える。**

🌐 [한국어](README.ko.md) | [English](../README.md) | [日本語](#) | [中文](README.zh.md) | [Español](README.es.md) | [Français](README.fr.md)

---

BLIPは痕跡を残さない一時的なチャットサービスです。
アカウントも、履歴も、プロフィールもありません。リンク一つで始まり、終わればすべてが消えます。

> 「この会話は残す必要がない」— そんな瞬間のために作られました。

---

## なぜ BLIP？

既存のメッセンジャーは多くのものを残します。アカウント、友達リスト、チャット履歴、通知...
しかし、世の中には**少し話して消えればいい会話**の方がはるかに多いのです。

| 既存のメッセンジャー | BLIP |
|---|---|
| アカウント作成が必要 | アカウント不要 |
| チャット履歴を永久保存 | 履歴ゼロ — 復元不可能 |
| 友達追加が必要 | リンク一つで即参加 |
| サーバーにデータ保管 | サーバー保存なし、E2E暗号化 |

## コアコンセプト

- **準備コスト0** — リンク一つで即開始
- **持続性0** — 終わったら復元不可能
- **アイデンティティ0** — アカウント、友達、プロフィールなし
- **100%合意ベース** — 参加者全員が同意した瞬間だけ存在
- **自動シュレッダー** — 最新メッセージだけが画面に残り、古いメッセージはリアルタイムで破棄
- **キャプチャガード** — スクリーンショットや画面録画の試行を検知し、メッセージを即座にぼかし処理

## 使い方

```
1. ルーム作成  →  ボタン一つで作成
2. リンク共有  →  相手に送る
3. チャット    →  E2E暗号化リアルタイムチャット
4. 終了        →  全データ即時破棄
```

## ユースケース

- 「ちょっと話して部屋を爆破」
- 「作戦会議後に痕跡消去」
- 「リンク一つで即集合」
- ゲームパーティー調整、イベント現場の連絡、機密性の高い一回限りの会話

## フィロソフィー

BLIPはメッセンジャーではありません。
**使い捨ての通信ツール**です。

人を長く引き留めるために存在しません。
面倒を取り除き、話し、消えるために存在します。

### やらないこと

このサービスは意図的に以下を**行いません**：

- ~~友達追加~~
- ~~チャット履歴保存~~
- ~~ユーザープロフィール~~
- ~~会話アーカイブ~~
- ~~SNS化~~

> 利便性のために哲学を捨てることはありません。

## 技術スタック

- WebSocketベースのリアルタイム通信
- エンドツーエンド暗号化（E2E — Curve25519 ECDH + XSalsa20-Poly1305）
- サーバーは中継者の役割のみ
- ルーム終了時：サーバー・クライアント両方で復元不可能
- 自動シュレッダー：表示範囲外のメッセージはblob URL解放と共に即削除
- キャプチャ防止：タブ切替・ショートカット・右クリック検知でメッセージぼかし処理

## BLIP me — 使い捨て連絡リンク

プロフィールにリンクを1つ共有するだけ。誰かがクリックすると、即座に1:1暗号化チャットが始まります — アカウントもフレンド追加も不要。

- **自分だけのリンク** — ユニークなURLを作成（例: `blip.me/yourname`）
- **リアルタイム通知** — 訪問者が接続した瞬間に即通知
- **リンク管理** — いつでもURL変更・削除可能
- **アカウント不要** — デバイスベースのトークンで所有権を証明
- Webで `/blipme` にアクセス、モバイルで **BLIP me** タブを選択

## マイルーム

最近作成または参加したルームはローカルに自動保存されます — サーバー保存なし。

- **自動保存**: ルーム作成/参加時にlocalStorage（Web）またはSecureStorage（モバイル）に保存
- **ワンクリック再参加**: 保存済みパスワードで再入力なしに再入室
- **管理者権限の維持**: グループチャット管理者トークンが保存され権限を維持
- **統合リスト**: 1:1とグループチャットを一か所で管理
- Webで `/my-rooms` にアクセス、モバイルで**チャット**タブをタップ

> パスワードと管理者トークンはデバイスの外に出ません。ブラウザデータを削除すると永久に削除されます。

## 埋め込み

iframeひとつでどんなウェブサイトにもBLIPチャットを追加できます：

```html
<iframe
  src="https://blip-blip.vercel.app/embed"
  width="400"
  height="600"
  style="border: none;"
  allow="clipboard-write"
></iframe>
```

埋め込みからのイベントを受信：

```js
window.addEventListener('message', (e) => {
  if (e.origin !== 'https://blip-blip.vercel.app') return;

  switch (e.data.type) {
    case 'blip:ready':         // ウィジェット読み込み完了
    case 'blip:room-created':  // ルーム作成（roomId, shareUrl）
    case 'blip:room-joined':   // チャット入室
    case 'blip:room-destroyed': // ルーム破棄
  }
});
```

- 軽量レイアウト — 広告なし、ナビゲーションなし
- 完全なE2E暗号化を維持
- 共有リンクが埋め込みコンテキスト内で維持
- 完全な例：[embed-example.html](../web/public/embed-example.html)
